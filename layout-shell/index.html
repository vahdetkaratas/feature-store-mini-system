Portfolio · batch feature engineering pattern

Feature Store Mini System

Churn-style raw CSV in → one deterministic build → validated feature table out. Definitions, transforms, and checks live in code so training and batch scoring stay aligned.

  • 8derived features
  • 1pipeline entrypoint
  • 4validation checks

How it works

  1. 1 Definitions — each feature documents inputs, dtype, and meaning in one registry.
  2. 2 Transform — a single build step produces customer_id + feature columns.
  3. 3 Validate — duplicates, schema, and sanity checks before you trust the file.

Deliberately small: not Feast or Tecton — a reusable pattern for consistent offline features and clear interview narrative.

Run the pipeline

Same logic two ways: upload here (FastAPI) or run locally from the repo root.

After pip install -r requirements.txt:

python -m src.pipeline.build_feature_table

Stricter raw checks: python -m src.pipeline.build_feature_table --strict
